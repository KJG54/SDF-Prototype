# Phase 10: Memory / Lessons Learned

## Purpose

Record what should help future projects.

## Agent Responsibilities

- Create a final project memory packet.
- Propose global memory promotions for reusable lessons.
- Keep project memory separate from global memory.
- Update `PROJECT-CHECKLIST.md` and `project-checklist.json`.
- Update project status to the approved final state or next follow-up state.
- Ask approval before promoting durable lessons or preferences.

## Human Responsibilities

- Approve or reject proposed global memory updates.

## Required Output

- `MEMORY-001-project-memory-packet.json` / `MEMORY-001-project-memory-packet.md`
- Optional global memory proposals
- Phase gate
- Final phase summary
- Final status and checklist update

## Exit Criteria

Lessons are recorded in the right place, any global memory promotions are approved or rejected, and the project can be referenced later from its final status.

## Closeout Prompt

Summarize the lessons recorded, memory promotions accepted or rejected, final project state, remaining deferred work, and where future follow-up should begin. Then ask whether the human approves closing the project workflow or continuing with a new follow-up.

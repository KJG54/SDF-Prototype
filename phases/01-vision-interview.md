# Phase 1: Vision Interview

## Purpose

Understand what the human wants to build and why.

## Agent Responsibilities

- Ask questions until the project vision is clear enough.
- Suggest possible base, strong, and future versions.
- Separate user-requested scope from agent-suggested options.
- Surface risks, vague areas, unrealistic expectations, and useful tools.
- Ask what computer, operating system, and IDE/editor the human is using to develop the project.
- Ask where the finished project should run.
- Update `PROJECT-CHECKLIST.md` and `project-checklist.json`.

## Human Responsibilities

- Describe the desired project in plain language.
- Answer questions.
- Choose the initial version direction.

## Required Output

- `VISION-001-vision.md`
- `VISION-001-vision.json`
- Phase summary
- Phase gate
- Open questions
- Initial project record if the human wants to continue

## Required Intake Topics

- Development computer and operating system
- Preferred IDE/editor
- Ability to install development tools
- Target platforms for the finished project
- Responsive/device expectations for websites and web apps

## Exit Criteria

The human has selected a base v1, strong v1, or custom blend to refine into requirements.

The phase must close with a direct approval question before moving to requirements.

# Generic Project Playbook

Use this when no specialized playbook clearly fits.

## Ask Early

- What should the project do?
- Who uses it?
- What does success look like?
- What platforms matter?
- Does it need data storage?
- Does it need accounts, payments, AI, or external APIs?
- Should it be private, shared, or public?

## Stack Guidance

Choose the simplest stack that can satisfy the approved requirements and shipping target. Prefer common, maintainable technologies over exotic choices unless the project needs them.

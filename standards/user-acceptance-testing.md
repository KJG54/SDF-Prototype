# User Acceptance Testing Standard

User acceptance testing asks whether the project works for the human, not just whether technical checks pass.

Use this for apps, games, websites, tools, workflows, and anything with subjective behavior.

## What To Capture

- Workflow tested.
- Who tested it.
- Device, OS, browser, or runtime used.
- Steps followed.
- Expected result.
- Actual result.
- Pass/fail status.
- Human satisfaction notes.
- Issues found.
- Must-fix before review.
- Acceptable to defer.
- Retest result after fixes.

## Agent Responsibilities

- Give clear manual testing steps.
- Make the test small enough for the human to complete.
- Ask what felt wrong, confusing, slow, ugly, or missing.
- Record human feedback in a structured artifact.
- Do not exit Testing / Verification until must-fix issues are resolved or explicitly deferred.

## Human Responsibilities

- Try the workflow.
- Report what happened.
- Say whether the result is acceptable for the current version.

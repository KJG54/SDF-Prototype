# Coding Standard

- Follow the conventions of the generated project.
- Prefer clear, maintainable code over clever code.
- Add abstractions only when they reduce real complexity.
- Keep changes scoped to the approved task.
- Do not silently introduce unrelated refactors.
- Verify behavior with tests, builds, or manual checks appropriate to the project.
- Use Visual Studio Code as the default IDE/workspace unless the project has a strong reason to use another environment.
- Do not treat an IDE as a replacement for required compilers, SDKs, build tools, runtimes, or packaging tools.

# Documentation Standard

Documentation should help a human use, run, maintain, or understand the project.

Every shipped project should include:

- README
- setup instructions
- environment setup guide
- run instructions
- dependency notes
- known issues
- future roadmap or deferred ideas
- final verification summary

Human action instructions should be written for a beginner. Include exact folders, exact commands, why the action matters, how to verify success, and what to tell the agent afterward.

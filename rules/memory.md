# Memory Rules

Memory is a reference layer, not the core runtime.

## What Goes Into Memory

- session summaries
- stable user preferences
- framework decisions
- project history
- known problems
- reusable patterns
- curated reference notes

## Promotion Rule

Low-risk session summaries can be written when the human asks to wrap up. Durable reusable lessons, preferences, known problems, and framework guidance should be proposed and approved before becoming global memory.

## Reference Docs

Stored reference notes should include source links, retrieved date, version if known, key facts, gotchas, and recheck timing. Current official docs win over old memory when decisions depend on up-to-date facts.
